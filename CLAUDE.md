# teamsnelgrove.com

Personal blog, CV, contact. HTML generated by [Zine](https://zine-ssg.io).

## HTML

- Use semantic elements; never `<div>`/`<span>` when a semantic alternative exists.

## CSS

### Constraints

- Single file: `assets/style.css`. No additional stylesheets, `<style>` blocks, or `style=""` attributes.
- No external dependencies, no `@import`, no font services. System stacks or self-hosted in `assets/`.
- No build step. No PostCSS/Sass/Tailwind.
- Evergreen browsers only — no vendor prefixes, polyfills, or legacy fallbacks.
- No dark mode, no print styles.
- Use modern CSS freely: nesting, `:where()`/`:is()`/`:has()`, logical properties, `clamp()`/`min()`/`max()`, container queries, `text-wrap: balance/pretty`, `color-mix()`.

### Layout

- Flexbox/Grid only. No floats; absolute positioning only for genuinely overlaid elements.
- Single centered column. Constrain reading width via `max-width` on `main`/`article`/`section`.
- Mobile-first. Prefer intrinsic sizing (`clamp()`, `min()`, `auto-fit minmax`) over media queries. Add a viewport breakpoint only when intrinsic sizing can't express the change.

### Tokens

- Define on `:root`: colors (`--color-bg`, `--color-text`, `--color-heading`, `--color-link`, `--color-muted`, `--color-rule`, `--color-code-bg`), spacing (`--space-1`–`--space-6`, fluid), type (`--text-sm`–`--text-2xl`, fluid), `--font-serif`, `--font-mono`, `--measure`, `--radius`.
- Reference tokens in component rules; do not hardcode colors or spacing outside `:root`.

### Typography

- Serif body: `Georgia, "Times New Roman", serif`. Mono: `ui-monospace, ...`.
- Fluid type via `clamp()`. Body line-height 1.5; headings tighter (1.15–1.3).

### Selectors

- Class selectors for components; element selectors for base typography and reset.
- Native nesting, max 2 levels deep.
- `:where()` to keep base specificity low.
- No ID selectors for styling. No `!important`.

### Reset

Josh Comeau style at top of file: `box-sizing: border-box`, zero margins on text elements, `body { line-height: 1.5; -webkit-font-smoothing: antialiased; }`, block media with `max-width: 100%`, `font: inherit` on form controls, `overflow-wrap: break-word` on text, `text-wrap: balance` on headings, `text-wrap: pretty` on `p`.

### File order

1. Reset
2. `:root` tokens
3. Base elements
4. Layout primitives (`main`, `article`, `section`, `nav`, `footer`)
5. Components
6. Syntax highlighting

Keep component rules colocated.

### Syntax highlighting

- Zine uses [flow-syntax](https://github.com/neurocyte/flow-syntax) (tree-sitter). Each token gets a class from its capture name with dots → underscores (`punctuation.bracket` → `.punctuation_bracket`). Capture names are conventional across grammars — style by capture, not by language.
- Theme: Catppuccin Latte, mirroring [catppuccin/helix](https://github.com/catppuccin/helix/blob/main/themes/default/catppuccin_latte.toml). `--ctp-*` palette is scoped to `pre code` (code blocks only — not site chrome).
- Tokens may carry multiple classes (e.g. `class="error punctuation_bracket"`). `.error` is declared last so it wins.

### Aesthetic

Minimal, content-first (cf. andrewconner.com). Neutral bg, near-black text, restrained palette, serif body, generous whitespace, quiet rules. When in doubt, remove.

## Project structure

```
teamsnelgrove-site/
  zine.ziggy              # Site config
  content/                # SuperMD (.smd)
    index.smd             # Homepage
    about.smd
    blog/
      index.smd           # Blog section root
      first-post/
        index.smd         # Post with page assets
        fanzine.jpg
      second-post.smd     # Single-file post
  layouts/                # SuperHTML (.shtml)
    index.shtml
    page.shtml            # Generic page (about)
    post.shtml            # Blog post + prev/next
    blog.shtml            # Blog listing
    blog.xml              # RSS
    templates/
      base.shtml          # Inherited base
  assets/
    style.css
```
